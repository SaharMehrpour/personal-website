About Me

I am a researcher specializing in Software Engineering and Human-Computer Interaction (HCI). My work focuses on understanding how developers make and work with design decisions, and identifying the challenges they encounter in this process. I am dedicated to developing tools and approaches that assist developers in documenting, adhering to, and discovering design decisions.

I earned my Ph.D. in Computer Science from George Mason University in May 2024, under the supervision of Dr. Thomas LaToza in the Developer Experience Design Lab.

My Research

A Survey of Tool Support for Working with Design Decisions in Code (ACM CSUR 2023)

Developers make design decisions when choosing technical approaches, shaping software implementation and quality attributes. They work with these decisions by identifying, choosing, documenting, following, and understanding them. Design decisions include design rationale and rules. This article reviews research on managing design decisions in code, focusing on how tools can support developers. It discusses developers' goals, challenges, and potential tool support.

Sahar Mehrpour, Thomas D. LaToza.

DOI, Pre-Print, Supplementary Materials

Can static analysis tools find more defects? (EMSE 2023)

Static analysis tools check code for defects by comparing it against predefined rules. Previous studies have assessed these tools based on their ability to find known defects, but not their potential for future defect detection. To explore this potential, a study formulated code reviewer comments as rule violations and compared them to rules static analysis tools could check.

Sahar Mehrpour, Thomas D. LaToza.

DOI, Pre-Print, Dataset

Programming Tools for Working with Design Decisions in Code (PLATEAU 2021)

Design decisions are integral to coding, involving choices among alternatives. However, traditional documentation is often inadequate, being incomplete and outdated. To address this, active documentation is proposed, allowing for real-time checking and immediate feedback on violations. It includes explanations and code snippets to illustrate decisions. New methods for creating and editing checkable decisions are suggested to make active documentation practical.

Sahar Mehrpour, Thomas D. LaToza.

DOI, Pre-Print, Slides.

RulePad (ESEC/FSE 2020)

Effective documentation aims to aid developers in understanding design choices, yet it frequently becomes outdated and unreliable. A potential solution involves developers writing design rules that can check code for consistency. However, existing rule checkers often require learning specialized query languages or program analysis frameworks, which can be a barrier to creating project-specific rules. Two new authoring techniques for design rules have been introduced: snippet-based authoring and semi-natural-language authoring. These approaches, implemented in RulePad, enable developers to write and validate design rules more easily, offering a way to keep documentation up to date and reliable.

Sahar Mehrpour, Thomas D. LaToza, Hamed Sarvari.

Active Documentation (VL/HCC 2019)

Good documentation is crucial for developers to write code quickly and consistently with design decisions. However, traditional documentation is often disconnected from code. Active documentation proposes a method where design decisions are documented as rules and checked against the code. Developers can navigate to code examples in their project to understand and follow these rules. ActiveDocumentation is a tool that enforces these rules by checking code against AST queries, providing immediate feedback on violations and offering code snippets as examples for developers to follow.

Sahar Mehrpour, Thomas D. LaToza, Rahul K. Kindi.
